OSRS Plugin

Using Java, I developed a plugin for the video game "Old School Runescape." This plugin was developed using the tools provided by a third-party application that runs Old School Runescape called RuneLite. The plugin was designed to provide information and quality of life changes for a specific piece of content in the game, which is a raid to complete and receive a reward at the end. The plugin calculated points received in the raid while tracking lost points due to deaths, and all the while calculating the percentages of obtaining a rare item using Binomial Distribution via points received. The plugin was uploaded via a fork of the plugin hub from Runelite and after review from an official RuneLite developer, it was added swiftly to the application for any users to find and download.

AR Research Assistant Project

Using Unity and C# I created a small prototype of an Augmented Reality app, using the Azure kinect device and C# scripts in an Unity environment. The prototype used the Azure kinects advanced camera to gather information of a person and their relative joint position and rotation. The camera took this information and calculated it using built in software and was able to print the information in a format that Unity could understand using the built in Avatar Bone Structure. With this, I wrote a script that would translate in real time the movements of a person through the camera and map the bones and joints onto the model in the Unity environment. Thus creating a result of controlling an Avatars movement using real life movement.

AI Research Assistant Project

Using ROS (Robot Operating System) and an API for the Miro-e robot companion dog, we trained the AI of this robot to react to situations in the real world. The main interaction created was the ability for the robot to sense facial expressions. Using real world data to feed into the learning for the AI companion dog, it learnt what emotions looked like using cameras and act appropriately. Using the API coded in Java we coded how the robot should react to these expressions. If the person displayed a sad expression, the robot would come closer to the person to give comfort, and when it sensed it was being touched or petted, the robot dog would display a show of happiness and reassurance. With these interactions built, we demonstrated it to the university board members to have a showcase of our companion robot and display its interaction. Ending in a successful demonstration.

Classic Brick Breaker Game In Unity

Using the Unity environment and engine and C# scripts I created a practice game to learn more about unity and C#. The game is a take of the classic breakout game created from scratch on unity. The game contained two levels, a start screen, a gameover screen and a highscore tracker. I created a simple physics engine of the balls velocity and how it bounces off the paddle and it's launch trajectory in C#. From there, I created the bricks on unity and added interactions with the ball and the brick in C#, which would cause the brick to break displaying a breaking animation. The brick would occasionally spawn a power up in the form of an extra life. The game tracked a high score and would save previous highscore and challenge the player to break it, allowing the player to input their name once a new high score is obtained. The primary goal of this project was to learn more about Unity and C# scripts, to get a more indepth idea of how the language and engine works, to help create more projects in the future using more advanced techniques.

Java Platformer

Using only Java in the Eclipse IDE I programmed and developed a simple platformer consisting of a black square as the player and brick textured squares to act as platforms, including a moving platform. The player had the ability to move left and right, alongside jumping. To achieve this I created a physics based on the screen moving, while the player is always at it's center. With the platforms and the players, I have created collision detection which will tell the program that the player and the platform are colliding and for the player to be placed on top or against the platform, allowing it to be jumped on or walked on. The program features multiple classes such as, GameStateManager, Collision Detection, Level States, Panels, Maps, Applying Textures, and Player Physics. This program was developed to help further understand Java and create a platformer without using pre-built libraries or engines.
